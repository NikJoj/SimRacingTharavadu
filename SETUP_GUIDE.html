Setup Guide

Sim Racing Tharavadu — Live Data Configuration

This guide walks you through connecting the website to Google Sheets so that registrations are captured automatically and the leaderboard updates whenever your admins edit the spreadsheet. You need a free Google account. No coding experience is required.

⏱ Estimated time: 20–30 minutes, once.

2 Create the Apps Script (Registration Handler)

This small script receives form submissions from your website and writes them as rows in the Registrations sheet.

  1. In your Google Sheet, go to Extensions → Apps Script.
  2. Delete any placeholder code and paste the following:
const SHEET_NAME = "Registrations";

function doPost(e) {
  try {
    const data = JSON.parse(e.postData.contents);
    const sheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName(SHEET_NAME);

    sheet.appendRow([
      data.timestamp    || new Date().toISOString(),
      data.firstName    || "",
      data.lastName     || "",
      data.driverTag    || "",
      data.nationality  || "",
      data.email        || "",
      data.whatsapp     || "",
      data.discord      || "",
      data.platform     || "",
      data.wheel        || "",
      data.experience   || "",
      data.skillLevel   || "",
      data.carClass     || "",
      data.event        || ""
    ]);

    return ContentService
      .createTextOutput(JSON.stringify({ status: "ok" }))
      .setMimeType(ContentService.MimeType.JSON);

  } catch(err) {
    return ContentService
      .createTextOutput(JSON.stringify({ status: "error", message: err.message }))
      .setMimeType(ContentService.MimeType.JSON);
  }
}

// Optional: test this runs without error
function testSetup() {
  const sheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName(SHEET_NAME);
  Logger.log("Connected to sheet: " + sheet.getName());
}
  1. Click Save (the floppy disk icon or Ctrl+S). Name the project anything, e.g. SRT Registration Handler.
  2. Click Deploy → New deployment.
  3. Click the gear icon next to "Select type" and choose Web app.
  4. Set Execute as: Me and Who has access: Anyone.
  5. Click Deploy. Google will ask you to authorize — click through the permissions.

3 Paste the Apps Script URL into the Website

Open index.html in any text editor (Notepad, VS Code, etc.). Find the CONFIG block near the top — it looks like this:

const CONFIG = {
  APPS_SCRIPT_URL: "YOUR_APPS_SCRIPT_URL_HERE",
  ...
};

Replace "YOUR_APPS_SCRIPT_URL_HERE" with the URL you copied in Step 2. Keep the quotes.

4 Publish the Google Sheet as JSON

This lets the website read your Events and Leaderboard data live.

  1. In your Google Sheet, go to File → Share → Publish to web.
  2. In the first dropdown select Entire Document, in the second select Web page.
  3. Click Publish and confirm.
  4. Close that dialog — you don't need the URL it shows.

Now construct the JSON feed URLs manually using your sheet's ID. Your sheet URL looks like:
https://docs.google.com/spreadsheets/d/YOUR_SHEET_ID/edit

Build your two data URLs like this:

Events sheet URL:
https://docs.google.com/spreadsheets/d/YOUR_SHEET_ID/gviz/tq?tqx=out:json&sheet=Events

Leaderboard sheet URL:
https://docs.google.com/spreadsheets/d/YOUR_SHEET_ID/gviz/tq?tqx=out:json&sheet=Leaderboard

Replace YOUR_SHEET_ID with the long string of letters and numbers from your sheet's URL.

5 Add the Sheet URLs to the Website & Go Live

Back in index.html, fill in the rest of the CONFIG block:

const CONFIG = {
  APPS_SCRIPT_URL:      "https://script.google.com/macros/s/AKfyc.../exec",
  EVENTS_SHEET_URL:     "https://docs.google.com/spreadsheets/d/YOUR_SHEET_ID/gviz/tq?tqx=out:json&sheet=Events",
  LEADERBOARD_SHEET_URL:"https://docs.google.com/spreadsheets/d/YOUR_SHEET_ID/gviz/tq?tqx=out:json&sheet=Leaderboard",

  DEMO_MODE: false   // ← Change this to false!
};

Save the file. Open it in a browser and test a registration — you should see a new row appear in your Registrations sheet within seconds.

✓ The leaderboard updates automatically every time the page is loaded. No refresh mechanism needed — just edit the Leaderboard sheet and visitors will see the new data on their next page load.

7 Day-to-Day Admin Workflow

Adding a new event: Add a new row to the Events sheet with a unique id. Set status to upcoming. Save. Done — it appears on the site immediately.

Starting an event (going live): Change the event's status column from upcoming to ongoing. It will now appear in the Leaderboard tabs.

Updating standings: Edit the Leaderboard sheet — add or update rows with the correct eventId. Sort by pos ascending. The site always reads the latest data.

Closing an event: Change the status to completed (or delete the row). It will disappear from the Events grid and Leaderboard tabs.

Viewing registrations: Open the Registrations sheet anytime to see all submissions as a table. You can sort, filter, or export to CSV from there.

✓ Pro tip: Add a Google Sheets notification (Tools → Notification rules → "When a row is added") to get an email every time someone registers.
